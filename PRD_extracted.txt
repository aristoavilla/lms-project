PRODUCT REQUIREMENTS DOCUMENT (PRD)
1. Product Overview
Product Name
Saint Lucia School Learning Management System (Portfolio Project)
Purpose
Build a class-based Learning Management System (LMS) for a single school, supporting 9 homeroom classes (each of them 30 students) with structured role-based access control, assignments, grading, ranking, announcements, and attendance tracking.
Tech Stack
Frontend: React + Vite + TypeScript
Backend/DB/Auth: Convex
Data Fetching: React Query
API Layer: Convex client (no axios required)
File Storage: Convex file storage
State Updates: Optimistic updates enabled
Scope Constraints
Single school (the school has 9 classes, 3 classes (A, B, and C) for each grade)
Each class has its own management system
30 students each class
Academic portfolio-level production quality

6. Ranking Algorithm Specification
Subject Average
For each student: Average = Sum(subject assignment scores) / number of assignments
Overall Average
Average of all subject averages
Ranking
Sort descending by average score Tie breaker: earliest cumulative submission time OR alphabetical
